# コンセンサス論文フォーマット（校正基準）

本リポジトリで「コンセンサス論文」「コンセンサス・ステートメント」「エキスパート・コンセンサス・リポート」の校正・執筆を行う際の標準フォーマット。  
Cursor ルール `.cursor/rules/consensus-paper-editing.mdc` と併用する。

参照原稿: `content/drafts/ga_consensus/ga_consensus_draft.md`

---

## 全体構成

1. **大見出し（章タイトル）** — テーマを一言で示す日本語タイトル（例: `GAの問診と機能評価`）
2. **はじめに：`はじめに：◯◯`**
   - 1文目: 日常診療の現状・落とし穴
   - 2文目以降: 疾患の本質と臨床的・QOL上の意味
   - 最後: 「ここでは〜について解説する」でスコープ明示（2–4段落）
3. **Consensus statement ブロック**（2–4本が目安）
   - `## N. Consensus statement N`
   - 1–2文の**太字**コンセンサス文（である調）
   - `(Consensus Score = XX)`
   - `### 【解説】**サブタイトル**：対比・比喩`
   - 病態・エビデンス `[n]`、nAMD等との質的差異、症状・ADLの箇条書き
   - `### 【〇〇の注意点】`（各CSの直後に必須。〇〇は当該CSで扱う事項に合わせて命名。例：`【問診上の注意点】` `【画像診断上の注意点】` `【紹介上の注意点】`—全CSを問診に統一しない）
4. **Recommendations セクション**（1本）
   - `## Recommendations：短時間で共有可能な「◯◯フレーム」の利用`
   - 一般外来の時間制約を前提に3ステップ
   - 各ステップ: 概要1–2文 → 質問文・チェック項目（「〜ですか？」）
   - `### 【Recommendations（エキスパートパネルの推奨）】` 箇条書き
5. **ポイント** — `**ポイント：◯◯**`（鑑別・見逃し予防のキードライバー）
6. **おわりに：`おわりに：◯◯`**
   - 問診→検査→モニタリング→説明責任→パラダイムシフト
   - 締め: 受動的診療 vs 能動的診療
7. **参考文献** — 番号リスト、誌名イタリック、本文は `[1]` 形式

## フォーマット細則

| 項目 | 規則 |
|------|------|
| 略語初出 | 地図状萎縮（Geographic Atrophy；GA） |
| 以降 | GA, nAMD, BCVA 等（BCVA初出は矯正視力（BCVA: best-corrected visual acuity）） |
| 文体 | である調。「我々」「眼科医」「多くの患者」 |
| 箇条書き | `*   **ラベル**: 説明。「患者の声」` |
| 区切り | 大セクション間に `---` |
| 引用 | 上付き不可。`[1]` `[2,3]` |
| 用語（GAシリーズ） | 血管アーケード／neovascular AMD；nAMD／網状偽ドルーゼン（RPD）／hyper-autofluorescence（hyper-AF）／ペグセタコプラン（PEG）・アバシンカプタド ペゴル（ACP）初出→以降略語 |
| Recommendations 箇条書き | 患者への問いかけのみ「…ですか？」形式。読影・施設チェックは「」なしの確認文（例：OCTでiRORA/nGAの有無を確認する） |

---

## 内容の型（6ステップ）

1. 現状認識・問題提起（見逃し・意識の欠如・乖離）
2. 病態・エビデンス要約
3. 患者の困りごと（読書・運転・顔認識・家事・趣味）
4. 実践フレーム（3ステップ）
5. 鑑別キーポイント（時間軸、nAMDとの違い等）
6. パラダイムシフトで締める

## 校正チェックリスト

- [ ] 大見出しは日本語一言タイトルか
- [ ] はじめには3段落構成か
- [ ] 各 CS に Score プレースホルダーがあるか
- [ ] 【解説】サブタイトルに比喩＋対比があるか
- [ ] 各CS直後に `### 【〇〇の注意点】` があり、〇〇が当該CSのテーマと一致しているか
- [ ] Recommendations が3ステップ＋具体質問か
- [ ] ポイントセクションがあるか
- [ ] おわりにで受動 vs 能動の対比があるか
- [ ] です・ます調が残っていないか
- [ ] 引用が `[n]` 形式か
- [ ] 参考文献5–10本程度、誌名イタリックか
